# RoArm-M2-S 机械臂技术文档

> v1 SmartRobotArm 使用的 Waveshare RoArm-M2-S 4-DOF 桌面机械臂
> 本文档所有命令和协议细节都经过本项目实际验证,不是 wiki 直译。
>
> 官方 wiki: https://www.waveshare.com/wiki/RoArm-M2-S
> 官方 ROS 2 workspace: https://github.com/waveshareteam/roarm_ws_em0

---

## 1. 硬件规格

| 项 | 规格 |
|---|---|
| DOF | 4 个 revolute joints + 1 个夹爪关节 |
| 控制器 | ESP32 主控,自带 WiFi / USB serial |
| 伺服 | 4 × **Waveshare ST3215** 总线舵机(0.33 N·m 堵转,~4.71 rad/s 空载) |
| 工作半径 | ~280 mm(水平伸展) |
| 载荷 | ~500 g |
| 自重 | ~826 g |
| 连接 | USB type-C(串口) + 独立 12V 2A 电源 |
| 底座 | 87 × 91 mm |

### 关节结构(从底到末端)

| # | URDF 关节名 | 硬件含义 | URDF 限位(rad) | 备注 |
|---|---|---|---|---|
| 1 | `base_link_to_link1` | 底座偏航 (base yaw) | `[-π, π]` | 轴向 `[0, 0, -1]`,硬件正方向与 URDF 反向 |
| 2 | `link1_to_link2` | 肩部俯仰 (shoulder pitch) | `[-π/2, π/2]` | 硬件正方向与 URDF 反向 |
| 3 | `link2_to_link3` | 肘部俯仰 (elbow pitch) | `[-1.0, π]` | 硬件与 URDF 同向 |
| 4 | `link3_to_gripper_link` | 夹爪开合 | `[0, 1.5]` | **0 = 全闭, 1.5 rad ≈ 85.9° = 全开**;硬件帧偏移 π |

**重要**:ST3215 物理行程 0°–300°,但 URDF 里是通过以上偏移/反向折算到 ROS 标准的正方向,别乱改 URDF 限位。

## 2. 物理连接

```
RoArm-M2-S ──USB-C 线── Desktop PC (Windows 11)
                             │
                             └── usbipd-win 转发 ──→ WSL2 Ubuntu 22.04
                                                       └── /dev/ttyUSB0

RoArm-M2-S ──DC 12V 2A 独立电源── 墙插
```

重点:
- **USB 只负责数据,12V 电源必须独立接** —— 只插 USB 时 ESP32 能上电回命令,但舵机没电,发 `T:102` 不会动。
- 机械臂用 C 形夹固定在工作台右边缘,避免运动时底座移位破坏 eye-to-hand 标定。

## 3. Windows → WSL2 USB 转发(usbipd-win)

### 3.1 一次性安装

在 Windows PowerShell(管理员)运行一次:
```powershell
winget install --interactive --exact dorssel.usbipd-win
```

### 3.2 找到 RoArm 的 busid

```powershell
usbipd list
```
RoArm-M2-S 通常显示为 `USB-SERIAL CH340` 或 `Silicon Labs CP210x` 系列的串口设备。记下它的 `BUSID`(形如 `1-2`)。

### 3.3 第一次 bind(一次性)

```powershell
usbipd bind --busid <ROARM_BUSID>
```

### 3.4 每次 Windows 重启后 attach

```powershell
usbipd attach --wsl --busid <ROARM_BUSID>
```

项目里有 `scripts/attach_devices.bat` 可以一键跑:编辑第一行 `set ROARM_BUSID=...` 填入实际 busid,然后把文件放到桌面,重启后双击即可。

### 3.5 WSL2 内验证设备可见

```bash
ls /dev/ttyUSB* /dev/ttyACM*
# 成功的样子:
# /dev/ttyUSB0
```

如果看不到,排查顺序:Windows 端是否先 `usbipd attach` → Windows 端设备管理器里有没有 CH340 驱动 → WSL 内 `dmesg | tail` 看有没有 USB 相关日志。

### 3.6 解决串口权限问题

第一次打开时可能 `Permission denied`。把用户加入 `dialout` 组:
```bash
sudo usermod -aG dialout $USER
# 加完必须重开 shell,或者 newgrp dialout
groups | grep dialout
```

临时办法(重启 WSL 后失效):`sudo chmod 666 /dev/ttyUSB0`

## 4. USB 串口协议(Waveshare JSON T-code)

### 4.1 连接参数

| 项 | 值 |
|---|---|
| 波特率 | **115200** |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | None |
| 帧格式 | 每条指令 = 一行 JSON + `\n` |

### 4.2 我们使用的 T-code(已实战验证过的那些)

> 完整 T-code 列表见 Waveshare wiki,下面只列本项目 `arm_driver` 用到的几条。

#### T:605 — 启动/模式复位

```json
{"T": 605, "cmd": 0}
```
`arm_driver_node` 启动时会自动发一次,对应上游官方 driver 的同一行为。

#### T:102 — 设置所有关节角度

```json
{"T": 102, "base": 0.0, "shoulder": 0.0, "elbow": 1.57, "hand": 2.39, "spd": 0, "acc": 10}
```

| 字段 | 说明 |
|---|---|
| `base` | 底座偏航,弧度,**硬件帧(非 URDF!)** |
| `shoulder` | 肩部,弧度,**硬件帧** |
| `elbow` | 肘部,弧度,**硬件帧** |
| `hand` | 夹爪,弧度,**硬件帧**(注意 `hand = π - urdf_gripper`) |
| `spd` | 速度,`0` = 使用舵机默认速度 |
| `acc` | 加速度,建议 `10` 保证启动平滑 |

**硬件帧 vs URDF 帧换算**(已在 `arm_driver/roarm_protocol.py::urdf_to_hw` 封装):

```
hw.base     = -urdf.base_link_to_link1
hw.shoulder = -urdf.link1_to_link2
hw.elbow    = +urdf.link2_to_link3
hw.hand     = π - urdf.link3_to_gripper_link
```

所以在 ROS 2 层永远发 URDF 正方向,转换只在 driver 最底层发生。

#### T:105 — 查询当前状态

```json
{"T": 105}
```
机械臂会返回一行 `T:1051` 响应(见 4.3)。

#### T:1051 — 状态响应(arm → host)

```json
{"T": 1051, "x": 280.1, "y": 0.0, "z": 235.7, "b": 0.0, "s": 0.0, "e": 1.57, "t": 2.39, "torB": 0, "torS": 0, "torE": 0, "torH": 0}
```

| 字段 | 说明 |
|---|---|
| `x` / `y` / `z` | TCP 末端笛卡尔位置,**单位 mm**(driver 自动除以 1000 转成 m) |
| `b` / `s` / `e` / `t` | 四个关节当前角度,**硬件帧**,单位 rad |
| `torB` / `torS` / `torE` / `torH` | 每个关节的当前扭矩反馈(原始单位,直接 pass-through) |

读回来时 driver 自动做 `hw_to_urdf(...)` 转回 URDF 正方向后再发布到 `/joint_states`。

#### T:114 — LED 控制

```json
{"T": 114, "led": 0.8}
```
`led` 值范围 `0.0`–`1.0`(或整数 0–255)。用于视觉调试的标识灯。

### 4.3 独立探活(跳过 ROS 直接试协议)

```bash
srarm
python -m arm_driver.roarm_protocol            # 默认 /dev/ttyUSB0
python -m arm_driver.roarm_protocol /dev/ttyACM0  # 自定义端口
```
输出样例:
```
[probe] opening /dev/ttyUSB0 @ 115200
[probe] TCP (m): x=0.2801 y=0.0000 z=0.2357
[probe] joints (URDF rad): base=+0.000  sh=+0.000  el=+1.571  hand=+0.750
[probe] torque: B=0 S=0 E=0 H=0
```
没有这个输出 = 协议/线序/电源有问题,**必须先把这一步跑通再去碰 ROS**。

## 5. ROS 2 Humble 集成(SmartRobotArm)

### 5.1 `arm_driver` 节点接口

| 实体 | 类型 | 方向 | 用途 |
|---|---|---|---|
| `/joint_states` | `sensor_msgs/msg/JointState` | **pub** @ 20 Hz | 真机当前关节角 + 扭矩反馈(URDF 帧) |
| `/joint_command` | `sensor_msgs/msg/JointState` | **sub** | 目标关节角(URDF 帧),支持部分关节(未指定的保留上次命令) |
| `/gripper_command` | `std_msgs/msg/Float32` | **sub** | 只改夹爪:0.0 = 闭, 1.5 = 全开 |

**参数**:
| 名 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `serial_port` | string | `/dev/ttyUSB0` | |
| `baud_rate` | int | `115200` | |
| `publish_rate` | double | `20.0` | `/joint_states` 发布频率 Hz |
| `send_on_init` | bool | `true` | 启动时发 `T:605` |

**关节名**(必须用这 4 个,与 `soma_robot.urdf.xacro` 对齐):
```
base_link_to_link1
link1_to_link2
link2_to_link3
link3_to_gripper_link
```

### 5.2 单独启动 arm_driver

```bash
srarm
ros2 launch arm_driver arm_driver.launch.py
# 或者指定端口
ros2 launch arm_driver arm_driver.launch.py serial_port:=/dev/ttyACM0
```

### 5.3 验证真机响应

```bash
# Terminal A: 起 driver
srarm && ros2 launch arm_driver arm_driver.launch.py

# Terminal B: 观察真机关节角
srarm && ros2 topic echo /joint_states --once

# Terminal B: 让 base 转 10 度(0.17 rad)
ros2 topic pub --once /joint_command sensor_msgs/msg/JointState \
  '{name: ["base_link_to_link1","link1_to_link2","link2_to_link3","link3_to_gripper_link"], position: [0.17, 0.0, 1.5708, 0.75]}'

# Terminal B: 只开/关夹爪
ros2 topic pub --once /gripper_command std_msgs/msg/Float32 '{data: 0.0}'   # 闭合
ros2 topic pub --once /gripper_command std_msgs/msg/Float32 '{data: 1.5}'   # 全开
```

### 5.4 手柄遥操(`arm_teleop`)

**一键起所有东西**:
```bash
srarm
ros2 launch arm_teleop teleop.launch.py
```
这条命令同时启动 `arm_driver_node` + `gamepad_teleop_node`。

**PDP Xbox 手柄映射**(50 Hz 闭环):

| 手柄输入 | 作用 |
|---|---|
| 左摇杆 X | 底座偏航 (base_link_to_link1) 速度 |
| 左摇杆 Y | 肩部俯仰 (link1_to_link2) 速度 |
| 右摇杆 X | 肘部俯仰 (link2_to_link3) 速度 |
| 右摇杆 Y | 手腕 (link3_to_gripper_link) 速度 |
| LB | 夹爪往闭合方向移动 |
| RB | 夹爪往张开方向移动 |
| LT 模拟扳机 | 闭合速度倍率(按住 LB 时生效) |
| RT 模拟扳机 | 张开速度倍率(按住 RB 时生效) |
| Start | 回 Home 姿态(`[0, 0, π/2, 0.75]`)并解除急停 |
| Back / Select | **急停**(冻结当前姿态,忽略后续输入) |

死区 `|axis| < 0.15`,每个关节的最大速度:
- base: 1.2 rad/s
- shoulder: 0.9 rad/s
- elbow: 1.0 rad/s
- hand: 0.8 rad/s
- gripper 按键: 1.0 rad/s,扳机最大再乘 2.5

这些常量在 [gamepad_teleop_node.py](../src/arm_teleop/arm_teleop/gamepad_teleop_node.py) 顶部,需要改速度感直接改数值。

### 5.5 手柄 axis / button 映射的校准

如果发现摇杆对应的关节错了,或者按键作用不对,先跑:
```bash
sudo apt install -y joystick
jstest /dev/input/js0
```
观察每一根摇杆 / 每一个按键对应的 **axis 编号 / button 编号**,然后在启动时用 ROS 参数 override,比如:

```bash
ros2 run arm_teleop gamepad_teleop_node --ros-args \
  -p axis_left_x:=0 -p axis_left_y:=1 \
  -p axis_right_x:=3 -p axis_right_y:=4 \
  -p axis_lt:=2 -p axis_rt:=5 \
  -p button_lb:=4 -p button_rb:=5 \
  -p button_start:=7 -p button_select:=6
```

## 6. 安全操作手册

**开机顺序**(每次从冷状态):
1. 12V 电源先上电 —— 舵机会做一次自检回零
2. USB 连接 PC
3. Windows 端 `usbipd attach`
4. WSL 端跑 `ls /dev/ttyUSB*` 验证
5. 先跑 `python -m arm_driver.roarm_protocol` 探活
6. 再起 ROS 2 driver
7. 最后起 teleop

**关机顺序**:
1. Ctrl-C 停 teleop(急停按键更快)
2. Ctrl-C 停 driver
3. 拔 USB
4. 断 12V 电源

**紧急情况**:
- **按 Back/Select 按钮**:立刻急停(target 冻结,不再发命令)
- **Ctrl-C driver terminal**:driver 会关闭串口,舵机保持当前角度(不会突然回零)
- **直接断 12V 电源**:舵机失去扭矩,手臂会在重力下慢慢下落 — **要提前用手扶住**

**首次真机运行**:
- 第一次发 `T:102` 命令时,**用手扶住机械臂**
- 先发非常小的角度差(比如 base 转 5°),看方向是否正确
- 方向不对 → 检查 URDF ↔ 硬件帧转换(别改 URDF,改 `roarm_protocol.py::urdf_to_hw` 的符号)
